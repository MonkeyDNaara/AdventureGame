# Codex Adventure Version

This folder is a separate working copy. The original files in `/Users/ericwilts/AdventureGame` were left untouched.

## What changed

- The 7-point starting allocation was removed again.
- The player now starts with fixed HP, STR, and ARMOR values.
- HP is still displayed with `●` and `○`, based on current health.
- The status block stays above the map and shows `LVL`, HP, STR, ARMOR, plus an XP progress bar to the next level.
- Movement is designed around W A S D. The older direction words still work internally.
- The console screen is cleared before each redraw, so the old map is no longer visible after movement.
- Story text types more slowly and now pauses longer between paragraphs.
- Empty rooms no longer repeat the old default room text.
- The game starts with a welcome screen: press `i` for instructions or Enter to begin.
- The player can see the distant torch before getting normal vision; the torch lights a 1-tile area.
- Walking onto an item no longer picks it up automatically. Press Space while standing on it.
- Before combat, the enemy prompt shows level, HP, and STR once, then asks `Fight or flee?` with spacing.
- During combat, the map is hidden. Monster stats are shown on the left, player stats and inventory on the right.
- Potion use consumes the player's turn, then the monster attacks.
- Winning a fight logs `You won the fight! You gained xy XP.` and removes normal enemies from the maze.

## Useful entry points for a GUI

- `Maze.render(player)` returns the currently visible maze as text.
- `Maze.render_grid(player)` returns a list of rows for drawing cells in a GUI.
- `Maze.move_player(player, direction, input_func=..., combat_action_provider=...)` moves or resolves a tile and returns an outcome dictionary.
- `Maze.pickup_current_item(player)` picks up the item under the player after Space is pressed.
- `Maze.get_current_item_prompt()` returns the item prompt shown below the map.
- `format_status(player)` returns the fixed player status display.
- `format_combat_view(player, enemy)` returns the two-column combat view.
- `consume_messages(player)` gives the GUI the messages generated by movement, items, and combat.
